# Automative-Dashboard
Developed an Automotive Dashboard System in Embedded C to simulate the functionality of a vehicle instrument cluster. The system monitors and displays essential vehicle parameters such as speed, fuel level, engine temperature, gear position, and warning indicators, providing real-time status updates similar to modern automotive dashboards.

Key Features:
- Vehicle speed monitoring and display
- Fuel level indication with low-fuel alerts
- Engine temperature monitoring and warnings
- Gear position display (P, R, N, D)
- Turn indicators and hazard light simulation
- Odometer and trip meter functionality
- Event logging and status tracking
- Real-time dashboard updates
- Modular and scalable software architecture

Technologies Used:
- Embedded C
- Data Structures
- State Machine Concepts
- Linux Development Environment
- GCC Compiler
- Makefile
- Git & GitHub

Learning Outcomes:
- Automotive embedded software development
- Event-driven programming and state management
- Real-time system design principles
- Modular software architecture
- Debugging and testing embedded applications
- Version control and collaborative development

This project demonstrates the implementation of an automotive instrument cluster by integrating multiple vehicle monitoring modules into a unified dashboard system. It showcases embedded programming concepts, real-time data processing, and software design practices commonly used in automotive applications.
